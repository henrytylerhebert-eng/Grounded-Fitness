# Naming Conventions

## Repository Language

Use clear operating-system language:

- Engine.
- Workflow.
- Segment.
- Review.
- Record.
- Measurement.
- Assumption.
- Decision.

Avoid premature software language unless code actually exists:

- Platform.
- Automation.
- AI agent.
- Integration.
- Pipeline.
- Production sync.

## Module Names

Approved module names:

- Member Value Engine.
- Local Authority Engine.
- Lead Nurture Engine.

Only Member Value Engine is active in the initial build.

## Segment Names

Use plain, human-readable segment names:

- Consistent.
- Active.
- At Risk.
- New.

Avoid labels that shame members:

- Lazy.
- Inactive failures.
- Dropouts.
- Bad attendance.
- Churn risks in member-facing language.

## Content Names

Use names that describe the job:

- Weekly programming summary.
- Training theme notes.
- Segment email draft.
- Human review checklist.
- Send record.
- Measurement record.

## Tone Conventions

Internal member emails:

- Coaching-led.
- Warm.
- Specific.
- Practical.
- Low-pressure.

Lead content:

- Informative.
- Soft.
- Local.
- Low-pressure.

Public content:

- Local.
- Human.
- Educational.
- Authority-building.

## Prohibited Language

Avoid:

- CrossFit gym.
- Box.
- WOD.
- RX.
- Elite fitness.
- Beast mode.
- No excuses.
- Gym-bro phrases.
- Hype claims.
- Guaranteed outcomes.
- Shame-based urgency.
- Medical or health promises.
