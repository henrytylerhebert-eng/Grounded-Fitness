# Future Development Roadmap

These modules are intentionally deferred.

They should influence architecture and data design, but they should not be included in the initial MVP.

The current MVP remains:

Weekly Programming Email -> Wodify Segmentation -> Member Value Emails

## Future Module 1: New Member Success Engine

Purpose: Increase first 90-day retention.

Potential features:

- First class trigger.
- First week trigger.
- First 10 visits trigger.
- First 30-day check-in.
- First benchmark tracking.
- First event participation.

Potential metrics:

- 30-day retention.
- 60-day retention.
- 90-day retention.
- Visit frequency during onboarding.

Status: Roadmap only.

## Architectural Requirement

The repository should be designed so these modules can be added later without redesigning:

- Core member records.
- Attendance records.
- Coach records.
- Milestone records.
- Recognition records.
- Referral records.
- Content records.
- Communication records.

The MVP should remain intentionally small.

The roadmap should remain visible.

Future modules should influence architecture but not current scope.
